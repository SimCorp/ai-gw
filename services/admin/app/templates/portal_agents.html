Agent Integration Guide

LangChain, LlamaIndex, OpenAI Agents SDK, and Claude Code — all pointed at the gateway.

Endpoints at a glance

http://localhost:8002/v1 — OpenAI-compatible (all models)
http://localhost:8002/anthropic — Anthropic-compatible (Claude + Claude Code)

Both accept Authorization: Bearer sk-… with your gateway key.

LangChain

Min version: langchain-openai>=0.1.0

Using ChatOpenAI (recommended for all models):

from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="claude-sonnet-4-6",
    openai_api_key="sk-YOUR-KEY-HERE",
    openai_api_base="http://localhost:8002/v1",
    temperature=0.2,
)

response = llm.invoke("Summarise the changes in this pull request: ...")
print(response.content)

Using ChatAnthropic (for Anthropic-specific features):

from langchain_anthropic import ChatAnthropic

llm = ChatAnthropic(
    model="claude-sonnet-4-6",
    anthropic_api_key="sk-YOUR-KEY-HERE",
    anthropic_api_url="http://localhost:8002/anthropic",
    max_tokens=4096,
)

RAG pipeline:

from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain.chains import RetrievalQA
from langchain_community.vectorstores import FAISS

llm = ChatOpenAI(
    model="claude-sonnet-4-6",
    openai_api_key="sk-YOUR-KEY-HERE",
    openai_api_base="http://localhost:8002/v1",
)
embeddings = OpenAIEmbeddings(
    openai_api_key="sk-YOUR-KEY-HERE",
    openai_api_base="http://localhost:8002/v1",
)
vectorstore = FAISS.load_local("my_index", embeddings)
qa_chain = RetrievalQA.from_chain_type(
    llm=llm, chain_type="stuff",
    retriever=vectorstore.as_retriever(search_kwargs={"k": 4}),
)
result = qa_chain.invoke({"query": "What is SimCorp's code review policy?"})
print(result["result"])
Pitfalls
Use openai_api_base, not base_url. base_url is silently ignored in older LangChain versions.
Missing key error? Set openai_api_key= on the constructor. Alternatively, set OPENAI_API_KEY env var to your gateway key.
Tool calling works with claude-sonnet-4-6 and claude-haiku-4-5 via OpenAI function-call wire format.

LlamaIndex

Min version: llama-index-llms-openai>=0.1.0

Using OpenAI LLM class:

from llama_index.llms.openai import OpenAI
from llama_index.core import Settings

llm = OpenAI(
    model="claude-sonnet-4-6",
    api_key="sk-YOUR-KEY-HERE",
    api_base="http://localhost:8002/v1",
    temperature=0.1,
    max_tokens=4096,
)
Settings.llm = llm

RAG with embeddings through the gateway:

from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings

Settings.llm = OpenAI(
    model="claude-sonnet-4-6",
    api_key="sk-YOUR-KEY-HERE",
    api_base="http://localhost:8002/v1",
)
Settings.embed_model = OpenAIEmbedding(
    model="text-embedding-3-small",
    api_key="sk-YOUR-KEY-HERE",
    api_base="http://localhost:8002/v1",
)

documents = SimpleDirectoryReader("./docs").load_data()
index = VectorStoreIndex.from_documents(documents)
response = index.as_query_engine().query("What auth methods does the gateway support?")
print(response)
Pitfalls
LlamaIndex uses api_base, not base_url or openai_api_base. The parameter name changed between 0.9 and 0.10 — check your installed version.
Trailing slash: http://localhost:8002/anthropic — no trailing slash required.
LlamaIndex < 0.10: use ServiceContext.from_defaults(llm=llm) instead of Settings.llm.
Tip: Create a second key labelled "embeddings" for separate cost attribution between embedding and LLM calls.

OpenAI Agents SDK

Min version: openai-agents>=0.0.7

Basic agent:

from agents import Agent, Runner, OpenAIChatCompletionsModel
from openai import AsyncOpenAI

gateway_client = AsyncOpenAI(
    api_key="sk-YOUR-KEY-HERE",
    base_url="http://localhost:8002/v1",
)
model = OpenAIChatCompletionsModel(
    model="claude-sonnet-4-6",
    openai_client=gateway_client,
)
agent = Agent(
    name="CodeReviewer",
    instructions="You are a senior engineer performing code reviews.",
    model=model,
)
result = Runner.run_sync(agent, "Review this function for security issues: ...")
print(result.final_output)

Agent with tools:

from agents import Agent, Runner, OpenAIChatCompletionsModel, function_tool
from openai import AsyncOpenAI

gateway_client = AsyncOpenAI(api_key="sk-YOUR-KEY-HERE", base_url="http://localhost:8002/v1")
model = OpenAIChatCompletionsModel(model="claude-sonnet-4-6", openai_client=gateway_client)

@function_tool
def search_codebase(query: str) -> str:
    """Search the codebase for relevant files and functions."""
    return f"Found: services/auth/app/validators/api_key.py matches '{query}'"

agent = Agent(name="Navigator", instructions="Find relevant code.", model=model, tools=[search_codebase])
result = Runner.run_sync(agent, "Where is API key validation implemented?")
print(result.final_output)

Multi-agent handoff (route cheap → powerful):

from agents import Agent, Runner, OpenAIChatCompletionsModel, handoff
from openai import AsyncOpenAI

client = AsyncOpenAI(api_key="sk-YOUR-KEY-HERE", base_url="http://localhost:8002/v1")

def make_model(m): return OpenAIChatCompletionsModel(model=m, openai_client=client)

triage = Agent(name="Triage", model=make_model("claude-haiku-4-5"),
               instructions="Classify the request as 'code', 'docs', or 'other'.")
coder  = Agent(name="Coder",  model=make_model("claude-sonnet-4-6"),
               instructions="Write production-quality code.")
triage.handoffs = [handoff(coder)]

result = Runner.run_sync(triage, "Write a Python email validator.")
print(result.final_output)
Pitfalls
base_url must include /v1. http://localhost:8002 alone causes 404 on /chat/completions.
Pass openai_client= to OpenAIChatCompletionsModel, not to Agent or Runner.
Tracing: The SDK sends traces to api.openai.com by default. This will silently fail. Disable with Runner.run_sync(..., trace=False) or configure a custom exporter.
Jupyter: Use Runner.run_sync() or wrap Runner.run() with asyncio.run().

Claude Code CLI

Min version: @anthropic-ai/claude-code latest

One-time setup:

# Install
npm install -g @anthropic-ai/claude-code

# Add to ~/.bashrc or ~/.zshrc
export ANTHROPIC_API_KEY=sk-YOUR-KEY-HERE
export ANTHROPIC_BASE_URL=http://localhost:8002/anthropic

# Verify
claude --version
claude "What model are you running on?"

From Python (subprocess):

import subprocess, os, json

result = subprocess.run(
    ["claude", "--output-format", "json", "Summarise this file", "main.py"],
    capture_output=True, text=True,
    env={**os.environ,
         "ANTHROPIC_API_KEY": "sk-YOUR-KEY-HERE",
         "ANTHROPIC_BASE_URL": "http://localhost:8002/anthropic"},
)
print(json.loads(result.stdout)["result"])

Project-wide config via CLAUDE.md:

# Place at repo root — Claude Code reads this automatically
# CLAUDE.md

This project uses the SimCorp AI Gateway.
Set ANTHROPIC_BASE_URL=http://localhost:8002/anthropic in your environment.
API keys: http://localhost:8005/portal/keys
Pitfalls
ANTHROPIC_BASE_URL must point to /anthropic, not /v1. Claude Code uses the Anthropic SDK wire protocol.
WSL vs Windows: In WSL use http://gateway:8002 (Docker Compose service name). On bare Windows host use http://localhost:8002.
Rate limits in CI: Create one key per pipeline (not one shared key) to avoid hitting team RPM limits across parallel jobs.
Caching benefit: Repeated identical prompts (same CI job on unchanged code) are served from gateway cache at zero provider cost.

End-to-end health check — run this before debugging framework issues

import httpx

resp = httpx.post(
    "http://localhost:8002/v1/chat/completions",
    headers={"Authorization": "Bearer sk-YOUR-KEY-HERE"},
    json={"model": "claude-haiku-4-5",
          "messages": [{"role": "user", "content": "ping"}],
          "max_tokens": 10},
    timeout=30.0,
)
if resp.status_code == 200:
    print("Gateway OK:", resp.json()["choices"][0]["message"]["content"])
elif resp.status_code == 401:
    print("Auth failed — check your key")
else:
    print(f"Unexpected {resp.status_code}:", resp.text)
