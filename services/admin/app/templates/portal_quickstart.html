Quick Start

Everything you need to make your first API call in under 5 minutes.

How the gateway works

Every request flows through three layers before reaching the AI provider:

Your code
  ↓
Auth service (:8001)   — validates your key, enforces rate limits
  ↓
Cache engine (:8002)   — checks Redis for cached responses (x-cache: HIT)
  ↓
LiteLLM proxy (:8003)  — routes to provider, injects provider API key
  ↓
Provider (Anthropic / Google / GitHub / Ollama)

OpenAI-compatible endpoint

http://localhost:8002/v1

All models, all frameworks with OpenAI support

Anthropic-compatible endpoint

http://localhost:8002/anthropic

Claude models, Anthropic SDK, Claude Code CLI

Your first API call

Replace sk-YOUR-KEY-HERE with your actual key from My Keys.

curl http://localhost:8002/v1/chat/completions \
  -H "Authorization: Bearer sk-YOUR-KEY-HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-haiku-4-5",
    "messages": [
      {"role": "user", "content": "Hello, which model are you?"}
    ]
  }'
from openai import OpenAI

client = OpenAI(
    api_key="sk-YOUR-KEY-HERE",
    base_url="http://localhost:8002/v1",
)

response = client.chat.completions.create(
    model="claude-haiku-4-5",
    messages=[
        {"role": "user", "content": "Hello, which model are you?"}
    ],
)

print(response.choices[0].message.content)
import anthropic

client = anthropic.Anthropic(
    api_key="sk-YOUR-KEY-HERE",
    base_url="http://localhost:8002/anthropic",
)

message = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Hello, which model are you?"}
    ],
)

print(message.content[0].text)

Note: point base_url at /anthropic, not /v1. The two endpoints use different wire protocols.

# Add to your ~/.bashrc or ~/.zshrc
export ANTHROPIC_API_KEY=sk-YOUR-KEY-HERE
export ANTHROPIC_BASE_URL=http://localhost:8002/anthropic

# Then use Claude Code normally
claude "Explain this codebase"

# Or pass inline (no env var needed)
ANTHROPIC_API_KEY=sk-YOUR-KEY-HERE \
ANTHROPIC_BASE_URL=http://localhost:8002/anthropic \
claude "Summarise the last 10 commits"
Streaming

All models support streaming via both endpoints.

from openai import OpenAI

client = OpenAI(api_key="sk-YOUR-KEY-HERE", base_url="http://localhost:8002/v1")

with client.chat.completions.stream(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": "Write a haiku about caching."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

Common errors & fixes
401 — Invalid or revoked API key
Check Authorization: Bearer sk-YOUR-KEY. Keys are case-sensitive; prefix must be sk- (lowercase).
Connection refused / ECONNREFUSED
Wrong port. The gateway listens on :8002. From inside Docker use http://gateway:8002; from host use http://localhost:8002.
404 on /v1/chat/completions
Your base_url must end with /v1 (OpenAI SDK) or /anthropic (Anthropic SDK). Do not append /v1 twice.
429 — Too Many Requests
You have hit your team's rate limit. Implement exponential backoff with Retry-After header, or request a limit increase from Platform Engineering.
400 — model not found
Use exact model IDs from the table above. claude-haiku-4-5 not haiku or claude-haiku.
