Connectivity
Agent Relay — WebSocket Tunnel

The Problem It Solves

Laptop-hosted agents cannot accept inbound connections from internal servers. The relay bridges the gap — the agent dials out, holds a WebSocket open, and the gateway forwards invocations through that connection.

Result: Zero inbound firewall rules needed

Invocation Timeout

300s
max per relay invocation before timeout

Protocol Flow

1 Agent registers with Identity Pool — receives relay_token
2 Agent opens persistent WebSocket to relay service
3 Relay token mapped to Redis for cross-service auth
4 Caller invokes agent — relay forwards over WebSocket
5 On disconnect: agent auto-unregistered, token cleaned up
